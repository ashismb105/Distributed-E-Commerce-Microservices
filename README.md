# Distributed-E-Commerce-Microservices
A highly scalable back-end infrastructure for a modern e-commerce platform. It breaks down a monolithic architecture into distinct, independently deployable microservices (Inventory, Order Processing, Payment, and Notification). The system relies on event sourcing and the Saga design pattern to maintain data consistency across distribute


An enterprise-grade event-driven e-commerce backend built with:

- Java 17
- Spring Boot 3
- Spring Cloud
- Apache Kafka
- PostgreSQL
- MongoDB
- Docker
- Kubernetes
- Resilience4j
- Spring Security (JWT)
- Prometheus
- Zipkin

## Modules

- Service Registry
- Config Server
- API Gateway
- Common Library
- Order Service
- Inventory Service
- Payment Service
- Notification Service

## Architecture

Event-Driven Microservices using Apache Kafka and Saga Pattern.
